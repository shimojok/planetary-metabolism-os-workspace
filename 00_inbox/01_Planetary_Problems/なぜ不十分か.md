# 🔴 なぜ不十分か

もし各方剤が

```json
{
  "id": "F001",
  "name": "...",
  "metabolic_pathway": "PATH_01"
}
```

だけだと、

- PATH_01の方剤は全て同一ベクトル
    
- PATH_02も同様
    
- 最大でも5クラスタしか発生しない
    
- 294処方の機能分化が消える
    

→ これは「分類」ではなく「PATHラベル分割」に過ぎません。

---

# ✅ 本当に必要なデータ

最低限必要なのは：

```json
{
  "id": "F001",
  "name": "...",
  "metabolic_pathway": "PATH_01",
  "components": {
      "葛根": {
          "phytochemicals": [...],
          "action_tags": [...],
          "role": "君"
      },
      ...
  }
}
```

---

# 🔷 理由

我々がやることは：

```
PATH × 作用タグ × 配伍重み
```

を

```
MBT15機能空間
```

へ射影することです。

したがって必要なのは：

|要素|必須か|理由|
|---|---|---|
|id|必須|識別子|
|name|推奨|可読性|
|metabolic_pathway|必須|基本射影|
|components|必須|差異生成|
|action_tags|強く推奨|機能重み付け|
|role（君臣佐使）|推奨|重み付け|

---

# 🔥 重要

もし components が無い場合：

→ 211処方は実質5タイプにしか分かれません。

それはMBT15設計の意義を失います。

---
