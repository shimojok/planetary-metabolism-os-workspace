### 🔷 現在のJSONデータ

```json
{
  "mbt_kampo_library": {
      "version": "...",
      ...
  }
}
```

となっているため、  **実データがネストされている可能性が高い**ということですね。

### 🔷 なぜオブジェクト形式は不利か

```json
{
  "F001": {...},
  "F002": {...}
}
```

この形式だと：

- 並び順保証が曖昧
- キー抽出処理が増える
- 将来の外部API統合時に整合が崩れる可能性
    
---

### 貴方が必要なのは、1～4のうちのどれですか？

### 1. 理想構造

```json
[
  {
    "id": "F001",
    "name": "...",
    "metabolic_pathway": "PATH_01",
    "components": {...}
  },
  ...
]
```

### 2. このフォーム

```json
[
  { "id": "F001", ... },
  { "id": "F002", ... },
  ...
]
```


### 3. 処方配列そのもの

```json
{
  "formulas": [ ... ]
}
```


### 4. 純粋な配列JSON

```python
formulas_list = list(raw["mbt_kampo_library"]["formulas"].values())
```

---

結論を明確にします。

▶ **必要なのは「④ 純粋な配列JSON」＝トップレベルが配列の形式です。**

---

# ✅ 採用形式（最終確定）

```json
[
  { "id": "F001", "name": "...", "metabolic_pathway": "PATH_01", ... },
  { "id": "F002", "name": "...", "metabolic_pathway": "PATH_03", ... },
  ...
]
```

つまり：

- トップレベルは **[]**
    
- 各要素が1方剤
    
- ネスト不要
    
- "formulas" キーも不要
    

---

# ❌ なぜ ③ は避けるか

```json
{
  "formulas": [ ... ]
}
```

技術的には問題ありませんが、

- 1階層余分
    
- sklearn処理時に毎回抽出処理が必要
    
- API連携時に変換処理が増える
    

---

# ❌ なぜ ①と②の違いは重要ではないか

①と②は実質同じです。  
キーが完全であれば②で十分です。

---
