### 2. Human-Soil Health Linkage の設計（Core Logic）


このモジュールは、以下の3つの「転移」を追跡し、その相関係数を算出します。

#### ① 機能性成分の転移 (Nutrient Density Link)

* **データ:** 土壌の微生物多様性（SHE™スコア）と、収穫物の抗酸化物質（ポリフェノール、ビタミン類等）の相関。
* **ロジック:** 微生物が生成する代謝産物が、植物の防御反応（二次代謝）をいかに活性化したかを定量化します。

#### ② 微生物相の転移 (Microbiome Link)

* **データ:** 農地の微生物組成データと、その産物を継続摂取した消費者の「腸内フローラ（HealthBookデータ）」の変化。
* **ロジック:** 物理的な菌の移動だけでなく、土壌由来の代謝物がヒトの腸内環境をどう最適化したかを解析します。

#### ③ バイタル指標の転移 (Clinical Link)

* **データ:** 再生型農産物の摂取量と、HealthBookが記録する「血液バイタル」「炎症マーカー（CRP等）」の推移。
* **ロジック:** 「土壌の改善速度」が「ヒトの未病改善速度」に与える寄与率を算出し、これを**Health-PBPE**として価値化します。

### 3. Python 実装：相関解析モジュール（Correlation Engine）

AgriWare™とHealthBookのデータベースをクロス参照するエンジンのプロトタイプです。

```python
class HealthLinkageEngine:
    def __init__(self, agri_db, healthbook_db):
        self.agri_db = agri_db          # 土壌・作物データ
        self.health_db = healthbook_db  # ヒトのバイタルデータ

    def analyze_health_contribution(self, user_id, farm_id):
        """
        特定の農地の産物がユーザーの健康に与えた寄与度を解析
        """
        # 1. 土壌のSHEスコアと作物の栄養価を取得
        soil_vital = self.agri_db.get_soil_history(farm_id)
        crop_nutrition = self.agri_db.get_crop_quality(farm_id)

        # 2. ユーザーの健康バイタル（HealthBook）を取得
        user_vitals = self.health_db.get_user_vitals(user_id)

        # 3. 相関演算 (Pearson correlation / Time-series analysis)
        # 土壌の改善とヒトの炎症マーカー低下のラグ相関を算出
        contribution_score = self._calculate_correlation(crop_nutrition, user_vitals)

        return {
            "user_id": user_id,
            "contribution_score": contribution_score,
            "health_pbpe_reward": contribution_score * 1.5  # 健康貢献への追加ボーナス
        }

```
