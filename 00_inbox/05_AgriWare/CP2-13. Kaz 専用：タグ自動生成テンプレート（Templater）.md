（例：`_templates/tag-header.md`）

---

## 📄 **Templater テンプレート内容（そのまま貼るだけ）**

```markdown
---
created: <% tp.date.now("YYYY-MM-DD HH:mm") %>
updated: <% tp.date.now("YYYY-MM-DD HH:mm") %>
tags:
  - <% tp.system.prompt("Top-level tag (例: AgriWare / HealthBook / MBT55 / Strategy / Research / Daily)") %>
  - <% tp.system.prompt("Function tag (例: chat / analysis / definition / model / algorithm / architecture / coding / formula / evidence)") %>
  - <% tp.system.prompt("Context tag (例: GHG / carbon / soil / microbe / metabolism / ecosystem / investment / policy / nutrition)") %>
---

# <% tp.file.title %>

## Summary
（このノートの要点を1〜3行で）

## Key Points
- 
- 
- 

## Related Notes
- 

## Raw Content
（ここから自由に書く）
```

# 🔍 このテンプレートが Kaz に最適な理由

## 1. **タグを“自動で”正しい構造に誘導する**
Kaz のタグ体系は三層構造：

- 階層タグ（AgriWare / HealthBook / MBT55 / Strategy …）
- 機能タグ（chat / analysis / model / coding …）
- 文脈タグ（GHG / soil / metabolism / investment …）

Templater が  
**「タグを入力するプロンプト」**を出すので、  
毎回迷わず正しいタグが付けられます。

---

## 2. **Kaz の自由度を損なわない**
Kaz のノートは型にはまらないため、  
テンプレートは **“枠だけ提供して、内容は自由”** にしています。

- Summary  
- Key Points  
- Related Notes  
- Raw Content（自由記述）

**→ 構造化と自由度の両立。**

---

## 3. **Dataview ダッシュボードと完全連携**
このテンプレートでタグを付けると、  
先ほど作った Dataview ダッシュボードが **自動で分類・集約**します。

- 今日のノート  
- AgriWare ノート  
- MBT55 ノート  
- AIチャットログ  
- 戦略ノート  
- 技術ノート  
- 数式ノート  
- GitHub構造ノート  

すべて自動で整理されます。

---

## 4. **Kaz の Daily Notes の“研究OS化”が完成する**
Kaz の Daily Notes は  
- チャット  
- 研究ログ  
- 数式  
- コード  
- 戦略  
- 定義書  
- PDF  
- 画像  
- 外部記事  
などが混在する **研究所レベルのノート**。

このテンプレートは  
**「Kaz の研究OSの入口」**として機能します。

# 🛠️ Obsidian での設定方法（簡単）

1. Obsidian → 設定 → **Templater**  
2. Templates フォルダを指定（例：`_templates`）  
3. 上記テンプレートを `_templates/tag-header.md` として保存  
4. 新規ノート作成時に  
   - コマンドパレット → “Templater: Insert template”  
   - またはホットキー設定
