# Dataview 設定（あなたの画面に沿った最適解）

---

## ① Warn on empty result（結果ゼロのとき警告）

**推奨：OFF**

### 理由  
Kaz のノート量では、Dataview クエリが 0 件になることは日常的。  
ON にすると、  
- ダッシュボードが警告だらけになる  
- 視認性が落ちる  
- スマホでの表示が煩雑になる  

---

## ② Render null as（null の表示）

**推奨：`-`（デフォルトのまま）**

### 理由  
- 空欄が「エラー」に見えない  
- Markdown 表が読みやすい  
- Dataview の TABLE が整う  

---

## ③ Automatic view refreshing（自動更新）

**推奨：OFF（強く推奨）**

### 理由  
Kaz のノート量では、ON にすると  
- ノート保存のたびに Dataview が全再計算  
- スマホで重くなる  
- フリーズする  
- 大規模ノートで落ちる  

**研究OSでは OFF が最適。**

---

## ④ Refresh interval（更新間隔）

**推奨：デフォルトの 2500 のまま**

### 理由  
自動更新を OFF にするため、ここは影響が少ない。  
デフォルトで問題なし。

---

## ⑤ Date format（日時フォーマット）

**推奨：`MMMM dd, yyyy`**

### 理由  
- Dataview の並び順が安定  
- 英語表記の方が Dataview と相性が良い  
- 研究ログの視認性が高い  

---

## ⑥ Date + time format（日時＋時間）

**推奨：`MMMM dd, yyyy - HH:mm`**

### 理由  
- Daily Notes の集約が読みやすい  
- 研究ログのタイムスタンプが明確  
- 並び順が安定  

---

# Advanced Tables（あなたの画面に沿った最適設定）

## 1. Bind enter to table navigation  
**推奨：OFF**

### 理由  
- Kaz のノートは **表の中で Enter を押して改行したい場面が多い**  
- ON にすると Enter が「次の行へ移動」になり、  
  - 数式  
  - コード  
  - モデル  
  - コメント  
  が書きにくくなる  
- 特に **AIとの対話ログや研究ノートでは Enter 改行が必須**

**→ OFF が最適。**

---

## 2. Bind tab to table navigation  
**推奨：ON**

### 理由  
- Tab でセル移動できると、表の編集が圧倒的に速くなる  
- AgriWare のパラメータ表  
- MBT55 の機能群表  
- 投資家向け比較表  
などで非常に便利  
- Tab は改行と衝突しないため、ON にしても問題なし

**→ ON が最適。**

---

## 3. Pad cell width using spaces  
**推奨：OFF**

### 理由  
- ON にすると、表のセル幅を揃えるためにスペースが大量に挿入される  
- 結果として  
  - GitHub 上で表が崩れる  
  - Markdown が汚れる  
  - Dataview の TABLE と相性が悪くなる  
- Kaz のノートは **数式・コード・テキストが混在するため、整形は逆に邪魔**

**→ OFF が最適。**

---

## 4. Show icon in sidebar  
**推奨：OFF（好みで ON でも可）**

### 理由  
- サイドバーにアイコンが増えると、スマホ画面が狭くなる  
- Kaz の Obsidian は  
  - Daily Notes  
  - Git  
  - Dataview  
  - Canvas  
  など重要アイコンが多いため、余計なアイコンは減らした方が良い  
- ただし、表を頻繁に編集する場合は ON でも良い

**→ 基本は OFF。必要なら ON。**

---

# Kaz の研究OSに最適化した最終設定まとめ

| 設定項目 | 推奨 | 理由 |
|---------|------|------|
| Bind enter | OFF | 改行が必要な場面が多い |
| Bind tab | ON | 表編集が高速になる |
| Pad cell width | OFF | Markdown が汚れない |
| Show icon | OFF | スマホ画面を圧迫しない |

- GitHub との互換性が **完璧**  
- Dataview の TABLE とも **相性が良い**  
- スマホでも **軽くて安定**

Kaz のように **研究・戦略・AI対話・数理モデル・コードを毎日大量に扱う人**にとって、  
この設定が最も効率的です。
