### 2. PBPE分配アルゴリズムの設計 (Distribution Logic)

分配は、収穫時またはMRV（計測・報告・検証）の確定時にトリガーされます。

#### ① 分配比率のロジック (Splitter Logic)

NotebookLMでの議論に基づき、以下の比率をスマートコントラクトにハードコード（またはガバナンスで設定）します。

* **生産者 (Farmer): 50%**
* 理由: 再生型農業の実効者への最大還元。


* **企業・購入者 (Corporate/Consumer): 30%**
* 理由: Scope 3削減実績（VIC）としての権利付与。


* **AGRIX再生基金 (Global Fund): 10%**
* 理由: 次の農家へのMBT55提供やR&D、地域インフラへの再投資。


* **プラットフォーム・集荷業者 (Hub/Operator): 10%**
* 理由: システム維持管理および流通コストの補填。



#### ② ステータス遷移プロトコル

トークンは以下の3段階を経て有効化されます。

1. **Pending（保留）:** AgriWare™が「予測」を出した段階。
2. **Verified（検証済）:** Safely Chain™がデータの真正性を確認した段階。
3. **Released（解放）:** 収穫・出荷が確認され、スマートコントラクトが各ウォレットへ送金した段階。

### 3. スマートコントラクト・擬似コード (Solidity)

実装の核心部分となるロジックです。

```solidity
contract PBPEPaymaster {
    // インパクト確定時に発火する分配関数
    function distributeImpactTokens(
        uint256 totalPBPE,
        address farmer,
        address corporate,
        address hub
    ) external onlyAgriWare {
        
        uint256 farmerShare = (totalPBPE * 50) / 100;
        uint256 corporateShare = (totalPBPE * 30) / 100;
        uint256 fundShare = (totalPBPE * 10) / 100;
        uint256 hubShare = totalPBPE - (farmerShare + corporateShare + fundShare);

        // トークンのミント（発行）と転送
        pbpeToken.mint(farmer, farmerShare);
        pbpeToken.mint(corporate, corporateShare);
        pbpeToken.mint(agrixFundAddress, fundShare);
        pbpeToken.mint(hub, hubShare);

        emit ImpactFinalized(totalPBPE, farmer, corporate);
    }
}

```
