# 1) 今回やったこと（PoC の実作業）

* `病気と食生活の関係シート.xlsx` を読み込み、シート構造を自動解析しました。
  → Excel 内には「●」で関係を示す形式の行列（疾病 × 項目）が含まれていることを確認しました。
* `問診表２.pdf` からテキスト抽出（質問の原文サンプルを取得）。
* 「●」マークを **重み 1（リスクあり）**、空白を **0（リスク無し）** とする簡易ルールで重み行列を作成。
  （注：Excel に数値重みが無く「●」で示されていたため、まずはこの単純二値マッピングで PoC を実装しました。）
* 単純スコアリング関数を作成（各疾病について `raw_score = Σ weight[q]*answer[q]`）。

  * 今回の PoC は **回答を二値（0/1）** と仮定しています（0＝該当しない、1＝該当する）。
* 得られた raw score を各疾病ごとの理論的 min/max（この二値仮定下）で正規化して `0–100` のスケールに変換。
* サンプル患者（キーワード一致による簡易回答割当）で計算を実行し、上位リスクを出力しました。
* 生成物（CSV / JSON / README）を `/mnt/data/healthbook_poc/` に保存しました。

# 4) 重要な【仮定】と【制限】

PoC は「概念実証（Proof-of-Concept）」です。実運用・臨床運用レベルには以下の点を改善する必要があります。

**仮定**

* 「●」は一律で +1 のリスク寄与（重み）と解釈しました。
* 全問の回答を **二値（0/1）** で扱いました（実際は頻度・程度・数値など多様）。
* 正規化は二値上の min/max に基づく簡易法（臨床分布を用いる方が望ましい）。

**制限**

* Excel に数値重みが無く「●」で表現されているため、重みの大小（例えば +2 より +1 を強く評価したい等）が反映されていません。
* 問診PDFから質問を自動抽出しましたが、PDFのレイアウトにより正確に質問ID化できない行が残っています。
* ラボ値・腸内細菌・代謝物などのオミクス情報は今回は組み込んでいません（次フェーズで統合可）。

---

# 5) 次に進めるための推奨改善（優先順）

1. **Excel のクリーン化（必須）**

   * 「病名」行のみ抽出、列ヘッダをすべて `question_id` に変換。
   * 各セルの「●」/「−」を **明示的な数値重み（例 +1 / -1）** に変換するルールを確定してください（あるいは私が推奨ルールで自動変換します）。
2. **問診のマスタ CSV 化（必須）**

   * `question_id, label_ja, type, options, category`（食習慣/生活習慣/症状/環境）を作成。PDF からの自動抽出は手作業でのクリーニングが必要です。
3. **回答のスケール化**（必須）

   * 頻度や程度（例：週3回、毎日、時々）を 0–3 や正負のスコアで扱うルール化。
4. **ラベルデータで ML 学習**（強く推奨）

   * 30万人の履歴データ（匿名化済み）を使い、ルールベース→機械学習で補正するステップを入れることで精度が大幅に上がります。
5. **中医学フェノタイプ・腸内細菌・代謝データの投入**（拡張）

   * MBT55 に基づく腸内データや代謝物データを特徴量化して ML モデルに投入します。
