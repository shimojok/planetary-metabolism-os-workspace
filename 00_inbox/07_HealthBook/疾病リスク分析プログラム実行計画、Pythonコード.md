# 疾病リスク分析プログラム実行計画

## 1. 目的と成果物

- 問診表200項目と「病気と食生活の関係」シートを統合し、137疾病リスクを定量化。
    
- 個人ごとのリスクレポートと改善提案を生成。
    
- 医療機関・投資家向けに説明可能なAIモデルを構築。
    

## 2. データモデル

- **Questions**: `question_id, text, type`
    
- **Responses**: `user_id, question_id, value`
    
- **DiseaseRules**: `disease_id, question_id, effect(+/-), weight`
    
- **DiseaseMeta**: `disease_id, name, category`
    

## 3. アルゴリズム

- ルールベーススコア: シートの●を重み化し合算。
    
- 機械学習補正: ロジスティック回帰/XGBoostで確率的リスクを算出。
    
- 最終スコア: ルール＋MLを加重平均。
    

## 4. 検証

- 過去30万人データで交差検証。
    
- AUC、感度・特異度を算出。
    
- SHAPで説明可能性を付与。
    

## 5. システム構成（Azure推奨）

- データ: Azure SQL, Data Lake
    
- 処理: Data Factory, Databricks
    
- モデル: Azure ML, AKS
    
- API: Azure Functions
    
- 可視化: Power BI
    

## 6. レポート

- 個人向け: リスクスコア、寄与因子、改善提案。
    
- 医師向け: 詳細解析、代謝経路可視化。
    

## 7. スケジュール

1. 要件定義（1ヶ月）
    
2. ルール化・プロトタイプ（2ヶ月）
    
3. ML開発・内部検証（3ヶ月）
    
4. PoC臨床検証（6ヶ月）
    
5. 本番化（4ヶ月）
    

## 8. リスクと対策

- データ品質不足 → 前処理強化
    
- バイアス → 多様コホート検証
    
- 規制 → 匿名化・同意管理
