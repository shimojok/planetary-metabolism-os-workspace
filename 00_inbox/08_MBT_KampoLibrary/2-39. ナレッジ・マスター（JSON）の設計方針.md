### 1. ナレッジ・マスター（JSON）の設計方針


このJSONは「辞書」であり、浜田先生の30年の臨床知見をデジタル化した「資産」です。

* **リスク項目の定義**: 「早食い」や「野菜不足」といった回答が、どの疾病にプラス（リスク増）またはマイナス（リスク減）に働くかを定義します。
* **下條・ファイトケミカル・タグ**: 各疾病に対し、「どの系統のファイトケミカル（水溶性/脂溶性）」がリスクを相殺できるかを属性として持たせます。

---

### 2. 統合推論エンジン（Python）の実装仕様

Pythonエンジンは「脳」として機能し、以下の動的なステップを実行します。

1. **データの読み込み**: JSON化された浜田・下條マスターをメモリにロードします。
2. **フェノタイプ・マッピング**: 200項目の問診回答を解析し、137疾病のベーススコアを算出。
3. **AGRIX ✕ 下條ブースト計算**: 摂取している作物の「質」が高い場合、その疾病が属する「防衛エリア（水溶性/脂溶性）」に応じたリスク軽減率を乗算します。
4. **MBT55代謝シミュレーション**: 腸内細菌がその成分を正しく活性化できるかを判定し、最終的な「個別化リスク」を出力します。

### 3. 社会実装用プロトタイプ・コード（ハイブリッド構成）

このコードがBioNexusのプラットフォームの心臓部となります。

```python
import json

class HealthBook_Precision_Engine:
    def __init__(self, master_data_path):
        # JSONからマスターデータを読み込み
        with open(master_data_path, 'r', encoding='utf-8') as f:
            self.master_data = json.load(f)

    def analyze(self, user_responses, food_quality_data):
        """
        user_responses: 問診のYes/No
        food_quality_data: AGRIXが測定した作物のファイトケミカル含有量
        """
        final_risks = {}

        for disease in self.master_data['Diseases']:
            disease_name = disease['name']
            # 1. 浜田式ベースリスク計算
            score = 0
            for factor in disease['risk_factors']:
                if user_responses.get(factor) == "YES":
                    score += 1  # 臨床データに基づく重み付け
            
            # 2. 下條式ファイトケミカル相殺 (Quality Boost)
            defense_type = disease['defense_type'] # 水溶性か脂溶性か
            mitigation_factor = food_quality_data.get(defense_type, 0.0)
            
            # 質が高いほどリスクスコアを下げる (例: 最大50%軽減)
            score = score * (1.0 - (mitigation_factor * 0.5))
            
            final_risks[disease_name] = round(score, 2)

        return sorted(final_risks.items(), key=lambda x: x[1], reverse=True)

# 実行イメージ
# 知識マスター(JSON)を読み込み、Pythonで演算
engine = HealthBook_Precision_Engine('BioNexus_Master_V2.json')
# AGRIXデータ: このユーザーが食べる野菜は水溶性成分の質が40%高い
agrix_input = {'hydrophilic': 0.4, 'lipophilic': 0.2}
result = engine.analyze(user_answers_200, agrix_input)

```
