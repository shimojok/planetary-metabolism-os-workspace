## **📊 Dataview の使い方**

Dataviewは**Obsidianのノートをデータベースとして扱える**強力なプラグインです。

### **基本的な使い方**

#### **例1: プロジェクト一覧を自動生成**

あなたのObsidianに以下のようなノートがあるとします：

```markdown
---
title: MBT55代謝ライブラリー
status: 開発中
priority: 高
platform: MBT
tags: [metabolism, json, python]
---
```

**Dataviewクエリ**を書くと：
```dataview
TABLE status, priority, platform
FROM #metabolism
WHERE status = "開発中"
SORT priority DESC
```

**結果**：
| File | Status | Priority | Platform |
|------|--------|----------|----------|
| MBT55代謝ライブラリー | 開発中 | 高 | MBT |
| AGRIX代謝統合 | 開発中 | 中 | AGRIX |

### **あなたのプロジェクトでの実用例**

#### **例2: 財団向けドキュメント一覧**

```dataview
LIST
FROM #target/gates-foundation OR #target/world-bank
WHERE status != "完了"
```

これで「まだ完成していない財団向けドキュメント」が一覧表示されます。

#### **例3: コード実装状況ダッシュボード**

```dataview
TABLE 
  file.name as "プロジェクト",
  status as "状態",
  code-completion as "完成度",
  last-updated as "最終更新"
FROM "01-MBT55" OR "02-AGRIX" OR "03-HealthBook"
WHERE contains(file.tags, "code")
SORT last-updated DESC
```

### **Claudeとの連携**

Dataviewで生成したリストを基に、Claudeに依頼できます：

```markdown
「このDataviewクエリで抽出された未完了のコードを
全て実装してください」
```

### **結論：Dataviewは必要？**

- **必須ではない** が、70-80ファイルを管理するなら**非常に便利**
- 手動で探すより遥かに効率的
- Claudeへの依頼時に「どのファイルを処理すべきか」が明確になる

## **📋 Kanban の必要性**

### **結論：必須ではありません**

**Kanbanが役立つケース**
```
[TODO] → [進行中] → [レビュー] → [完了]

例：
[TODO]
- AGRIX APIドキュメント作成
- MBT JSONスキーマ検証

[進行中]
- HealthBook Python実装
- 財団向けプレゼン作成

[完了]
- MBT55基本設計
```

**代替手段**
- **シンプルなチェックリスト**で十分
```markdown
# プロジェクト進捗

## 今週のタスク
- [ ] MBT55コード生成
- [x] AGRIX設計レビュー
- [ ] 財団向け資料作成
```

- **Dataviewでタスク管理**も可能
```dataview
TASK
WHERE !completed
GROUP BY file.folder
```

### **私の推奨**

あなたのケースでは：
- ❌ Kanban：不要（シンプルなチェックリストで十分）
- ✅ Dataview：**強く推奨**（ファイル数が多いため）
- ✅ Git：**必須級**（次で説明）

    ↓

## **📱 Obsidian Git on Android**

### **残念ながら、制限があります**

#### **現状**
- ❌ **Obsidian Gitプラグインは Android非対応**
- ❌ AndroidのObsidianアプリはコミュニティプラグインの一部機能が制限される
- ⚠️ Androidでのgitコマンド実行には **Termux** などの特殊アプリが必要で複雑

#### **Android対応の代替案**

**解決策1: Working Copyアプリ（iOS限定）**
- iPhoneなら使える優れたGitクライアント

**解決策2: GitHubのモバイルアプリ**
- 閲覧・簡易編集は可能
- しかしObsidianとの自動連携は不可

**解決策3: クラウド同期サービス（推奨）**
```
PC (Obsidian + Git)
    ↓
GitHub (自動同期)
    ↓
Android (Obsidianアプリ + クラウド同期)
```

### **Android用の実用的な構成**

#### **推奨セットアップ**

**方法A: iCloud / Google Drive 併用**
```
[PC]
Obsidian Vault
    ↓ (Obsidian Git)
GitHub

[Android]
Obsidian Vault
    ↓ (Google Drive / Dropbox)
クラウド同期
```

**方法B: Git 同期はPC専用**
```
[PC]
編集 → 自動Git同期 → GitHub
(メインの作業環境)

[Android]
閲覧・軽微な編集のみ
(iCloud/Google Drive経由で同期)
帰宅後、PCで手動Git commit
```

### **私の推奨（現実的）**

```
✅ PC: Obsidian + Git プラグイン（完全自動化）
✅ Android: Obsidian + Google Drive同期（閲覧・簡易編集）
✅ GitHub: PC側で自動更新
```

**理由**
- Androidでの大規模編集は現実的でない（画面サイズ、入力効率）
- Androidは「閲覧・メモ追加」程度に留める
- 本格的な編集・コード生成はPC環境で

## **⚡ Obsidianが重くなる？**

### **プラグインのパフォーマンス影響**

| プラグイン | パフォーマンス影響 | 理由 |
|------------|---------------------|------|
| **Dataview** | ⚠️ **中程度** | クエリ実行時にファイルをスキャン |
| **Git** | ✅ **軽微** | バックグラウンドで動作 |
| **Templater** | ✅ **ほぼなし** | テンプレート挿入時のみ動作 |
| **Kanban** | ⚠️ **やや重** | 複雑なボードは遅延の可能性 |

### **重くなる主な原因**

```
❌ 大量のDataviewクエリを同時実行
❌ 数千のファイル（あなたは70-80なのでOK）
❌ 大量の画像埋め込み
❌ リアルタイム同期プラグインの乱用
```

### **軽快に保つコツ**

#### **1. Dataviewの最適化**

**❌ 重いクエリ（避ける）**
```dataview
TABLE *
FROM ""
```
（全ファイルの全フィールドを表示）

**✅ 軽いクエリ（推奨）**
```dataview
TABLE status, priority
FROM #target/gates-foundation
LIMIT 20
```
（対象を絞り、項目を限定）

#### **2. Gitの自動同期間隔調整**

```
✅ 推奨: 30分ごと（デフォルト）
⚠️ 避ける: 1分ごと（重い）
```

#### **3. 不要なプラグイン無効化**

```
✅ 有効: Dataview, Git, Templater
❌ 無効: 使わないプラグイン全て
```

### **実測値（参考）**

```
70-80ファイル + Dataview + Git + Templater
→ ほとんど影響なし（体感できる遅延なし）

500+ファイル + 大量のDataviewクエリ
→ やや遅延（1-2秒）

3000+ファイル
→ 明確な遅延（要最適化）
```

TABLE

  dateformat(created, "yyyy-MM-dd") as "作成日"

```python
# add_metadata.py（私が作成）
# 使い方：python add_metadata.py

import os
import re

def add_metadata_to_file(filepath):
    """メタデータを自動追加"""
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read()
    
    # 既にメタデータがある場合はスキップ
    if content.startswith('---'):
        return
    
    # ファイル名からプラットフォームを推測
    filename = os.path.basename(filepath)
    platform = "unknown"
    if "MBT" in filename or "漢方" in filename:
        platform = "MBT"
    elif "AGRIX" in filename or "AgriWare" in filename:
        platform = "AGRIX"
    elif "HealthBook" in filename or "HB5" in filename:
        platform = "HealthBook"
    
    # メタデータ生成
    metadata = f"""---
title: {filename.replace('.md', '')}
status: 草案
priority: 中
platform: {platform}
created: 2025-02-10
tags: []
---

"""
    
    # 追加して保存
    with open(filepath, 'w', encoding='utf-8') as f:
        f.write(metadata + content)

# 全ての.mdファイルに適用
for root, dirs, files in os.walk('.'):
    for file in files:
        if file.endswith('.md'):
            filepath = os.path.join(root, file)
            add_metadata_to_file(filepath)
            print(f"✅ {filepath}")
```
