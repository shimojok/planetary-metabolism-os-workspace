これはブラウザの**キャッシュ**が原因です。

---

## 原因

GitHub Pagesは一度配信したファイルをCDN（コンテンツ配信ネットワーク）にキャッシュします。PDFのURLが同じままだと、古いキャッシュが表示され続けます。

---

## 解決方法

### 方法①：URLにバージョン番号を付ける（推奨）

`index.html` のPDFリンクを以下のように変更します：

```html
<!-- 変更前 -->
<a href="slides_en.pdf">

<!-- 変更後 -->
<a href="slides_en.pdf?v=2">
```

`?v=2` を付けるだけでブラウザが新しいファイルとして認識し、キャッシュを無視して最新版を取得します。次回更新時は `?v=3` にするだけでOKです。

---

### 方法②：しばらく待つ

GitHub PagesのCDNキャッシュは通常**10分〜1時間程度**で自動的に更新されます。急がない場合はそのまま待つだけでも解決します。

---

## index.html の修正手順

1. GitHubリポジトリで `index.html` を開く
2. 鉛筆アイコン（Edit）をクリック
3. `slides_en.pdf` を `slides_en.pdf?v=2` に置換（2箇所）
4. `slides_ja.pdf` を `slides_ja.pdf?v=2` に置換（2箇所）
5. 「Commit changes」

これで即座に最新PDFが表示されるようになります。
