## ① 成分 → PATHマッピング結果（メモ）

### これは何か？

**あなたのJSONの中で、すでに行われている判断を「人間が読める形」で書いたもの**です。

> 「どの成分が、どの代謝PATHに寄与しているか」

を**一覧化した説明資料**です。

### なぜ必要か？

- JSONが今アップロードできない
    
- しかし **クラスタリングの正当性は残したい**
    
- 将来「なぜこのPATHになった？」と聞かれた時の説明用
    

### 内容レベル（これで十分）

```text
【成分 → PATH対応ルール（抜粋）】

・アルカロイド類（エフェドリン、ベルベリン 等）
　→ PATH_01（放線菌・脱糖・水溶性成分活性化）

・サポニン類（ジンセノサイド、グリチルリチン 等）
　→ PATH_03（脂質代謝・吸収促進）

・精油・テルペン類（ジンゲロール、リモネン 等）
　→ PATH_04（芳香族分解・神経系）

・多糖類（βグルカン、アラビノガラクタン 等）
　→ PATH_05（プレバイオティクス）

・ミネラル類（Ca, Mg, Fe 等）
　→ PATH_02（還元・補正代謝）
```

✔ 箇条書き  
✔ 完全網羅でなくてよい  
✔ **「判断の思想」が伝わることが重要**

## ② 生成済み行列（もし作成済なら）

### これは何か？

今回、私が内部で使った **数理的中核**です。

```text
293方剤 × 5 PATH
```

の **数値行列の定義情報**です。

### 注意

👉 **npyファイルそのものは不要です。**  
👉 _「どう作ったか」_ が分かれば十分です。

### 最低限必要な内容（メモでOK）

```text
【PATH寄与行列 定義】

・行：方剤（F001〜F294、F030欠番）
・列：PATH_01〜PATH_05

各セルの値：
- components 内の phytochemicals を走査
- concentration_mg_per_g を平均値化
- 対応する metabolic_pathway に加算
- polysaccharides → PATH_05 に加算
- minerals → PATH_02 に加算
- 最終的に L2正規化
```

これがあれば：

- 再実行可能
    
- 再検証可能
    
- Azure / Microsoft にも説明可能
