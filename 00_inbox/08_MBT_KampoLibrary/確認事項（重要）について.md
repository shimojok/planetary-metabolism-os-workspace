確認事項（重要）について

成分ごとの「濃度平均」は既にJSON内に存在しますか？
例："mean_concentration" フィールド
それとも別ファイル統合が必要ですか？
これにより計算パイプラインが変わります。

----

## 回答：濃度平均（mean_concentration）は **現状JSON内に存在しません**

現在のJSONでは、濃度は範囲形式で記述されています：

```json
"concentration_mg_per_g": "2.5-5.0"  // 現状
```

必要な形式：
```json
"mean_concentration": 3.75,  // 平均値
"concentration_range": "2.5-5.0"  // 範囲も保持する場合
```

## 計算パイプラインの選択肢

### 選択肢A：JSON内で事前計算（推奨）
変換スクリプトで一括処理：
```python
def calculate_mean(concentration_str):
    if "-" in concentration_str:
        low, high = map(float, concentration_str.split("-"))
        return (low + high) / 2
    return float(concentration_str)
```

### 選択肢B：アプリケーション実行時に動的計算
その都度計算：
```python
# 使用時に計算
mean = (2.5 + 5.0) / 2  # 3.75
```

### 選択肢C：別ファイルで管理
- `concentrations.json` として別途管理
- メインJSONと結合して使用

## 推奨：選択肢A

**事前計算済みJSONを生成するスクリプト**をご用意します。これにより：

1. パフォーマンス向上（実行時計算不要）
2. データの一貫性確保
3. 他のシステムとの連携が容易
