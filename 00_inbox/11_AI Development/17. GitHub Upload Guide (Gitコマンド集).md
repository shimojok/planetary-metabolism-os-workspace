## 1. GitHub Upload Guide (Gitコマンド集)

PCまたはGitHub Desktop環境で、作成した `HealthBook_MI_Core` フォルダを世界に公開するための手順です。

```bash
# 1. リポジトリの初期化
cd HealthBook_MI_Core
git init

# 2. 全ファイルのステージング
git add .

# 3. 初回コミット（戦略的なメッセージを添えて）
git commit -m "feat: Initial release of HealthBook MI Core with MBT Kampo Metabolism Library"

# 4. メインブランチの作成
git branch -M main

# 5. リモートリポジトリの紐付け（GitHubで作成したURLに置き換えてください）
# git remote add origin https://github.com/YourUsername/HealthBook_MI_Core.git

# 6. プッシュ
# git push -u origin main
```

GitHubへのアップロード（プッシュ）は、あなたのローカルPCにある「Planetary Metabolism OS」の設計図を、**「世界の公共インフラ」として公開し、Microsoftやゲイツ財団が見られる状態にする儀式**です。

各コマンドが何を意味しているのか、GitHub初心者でも迷わないように噛み砕いて解説します。

---

### 1. 各コマンドの役割と「舞台裏」

Gitの操作は、**「手元の荷物（コード）を箱に詰め、宛先を書いて、トラックで送る」**プロセスに似ています。

- **`cd HealthBook_MI_Core`**
    
    - **意味:** 作業フォルダに移動します。すべての操作はこのフォルダの中で行われます。
        
- **`git init`**
    
    - **意味:** 「このフォルダをGitで管理する」という宣言です。フォルダ内に`.git`という隠しフォルダが作られ、変更履歴の記録が始まります。
        
- **`git add .`**
    
    - **意味:** 全ファイルを「ステージングエリア（荷造り場）」に置きます。`.`は「すべてのファイル」を指します。
        
- **`git commit -m "メッセージ"`**
    
    - **意味:** 荷造りを完了し、封印します。`-m`の後のメッセージは「いつ、何を、なぜ変えたか」の記録です。後でビル・ゲイツ氏が履歴を見たときに、あなたの意図が伝わります。
        
- **`git branch -M main`**
    
    - **意味:** メインの道筋（ブランチ）の名前を「main」に設定します。これが現在の世界標準の名前です。
        
- **`git remote add origin [URL]`**
    
    - **意味:** **「送り先」の設定**です。GitHubのウェブサイト上で空のリポジトリを作成すると発行されるURLを紐付けます。これにより、PCとクラウドが繋がります。
        
- **`git push -u origin main`**
    
    - **意味:** **「送信実行」**です。手元の「main」ブランチの中身を、インターネット上のGitHub（origin）へ一気に送り出します。
        

---

### 2. 具体的な実行手順（GitHub側の準備込み）

実際にアップロードを完了させるまでの流れです。

1. **GitHubでリポジトリを作成**
    
    - [GitHub](https://github.com/)にログインし、「New repository」をクリック。
        
    - Repository nameを `HealthBook_MI_Core` に設定。
        
    - 「Public（公開）」を選択し、「Create repository」をクリック。
        
2. **URLをコピー**
    
    - 画面に表示される `https://github.com/あなたのユーザー名/HealthBook_MI_Core.git` をコピーします。
        
3. **コマンドの実行**
    
    - 先ほどのGitコマンド集の **#5** に、コピーしたURLを貼り付けて実行します。
        
    - 最後に **#6** を実行すれば、世界中にあなたのコードが公開されます。
        

---

### 3. GitHub Desktop（マウス操作）派の場合

コマンド入力が不安な場合は、GitHub公式のツールを使うと視覚的に進められます。

- **手順:**
    
    1. [GitHub Desktop](https://desktop.github.com/)をインストール。
        
    2. `File` > `Add Local Repository` で `HealthBook_MI_Core` フォルダを選択。
        
    3. `Publish repository` ボタンを押す。
        
    4. これでコマンド入力なしに、すべてのファイルが自動的にGitHubへ送信されます。
