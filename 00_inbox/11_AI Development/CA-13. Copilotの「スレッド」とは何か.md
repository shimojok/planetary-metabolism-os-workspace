# ① Copilotの「スレッド」とは何か

GitHub Copilot や
Microsoft Copilot の会話スレッドは

基本的に

```
単なる対話コンテキスト
```

です。

つまり

```
Thread A
Thread B
Thread C
```

を作っても

```
記憶共有なし
自律判断なし
連携なし
```

になります。

これは **AIエージェントではありません**。

---

# ② AIエージェントの条件

一般的にエージェントには4要素があります。

```
① goal（目的）
② memory（記憶）
③ tools（外部ツール）
④ autonomy（自律性）
```

例えば

```
AGRIX Agent
```

なら

Goal

```
土壌と農業最適化
```

Tools

```
soil database
climate data
crop model
```

Memory

```
地域データ
農業履歴
```

これが **エージェント**です。

# ④ 本物の6エージェント構造

本来はこうです。

```
Orchestrator
   │
   ├ AGRIX Agent
   ├ MBT Agent
   ├ Health Agent
   ├ Data Agent
   └ Simulation Agent
```

ここで重要なのは

```
Agent → Agent call
```

です。

例

```
Health Agent
   ↓
MBT Agent に問い合わせ
   ↓
腸内菌解析
```
