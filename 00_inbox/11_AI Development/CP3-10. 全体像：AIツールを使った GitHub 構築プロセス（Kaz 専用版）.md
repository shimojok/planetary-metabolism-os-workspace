# 🔧 全体像：AIツールを使った GitHub 構築プロセス（Kaz 専用版）

あなたのプロジェクトは、一般的な AI プロジェクトではなく、

- MBT55（生態学・代謝学）
- AgriWare（農業・気候・経済）
- HealthBook（代謝不全解析）
- SafetyChain（食品・農業の安全性）
- Probiotics／Herbal／Food のスクリーニング
- グリーンプレミアム／GHG削減シミュレーション

という **巨大な知識体系 × モデル × データ × コード** を統合するものです。

そのため、構築プロセスは **5層構造** になります。

---

# 1️⃣ 構造化（Obsidian → JSON/YAML）

### 目的

Obsidian に散在するコード・ノート・モデル定義を、  
**AIが扱える構造化データに変換する**。

### 使うAIツール

- Claude（構造化・要約・変換）
- ClaudeCode（コード生成）
- Cursor（コード整理）

### ここで行うこと

- HealthBook の 294方剤を JSON に分割
- 137疾病マトリックスを JSON 化
- 200問診項目を JSON 化
- MBT55 代謝ロジックを YAML 化
- AgriWare のモデル仕様を YAML 化
- SafetyChain のロジックを JSON 化

### 結果

**/data フォルダにすべてのデータが整理される。**

---

# 2️⃣ データレイヤー構築（JSON → Python データクラス）

### 目的

JSON を読み込み、あなたのエンジン（HealthBook／AgriWare／SafetyChain）に渡せる形にする。

### 使うAIツール

- ClaudeCode（データローダー生成）
- Cursor（型定義・変換コード生成）

### ここで作るコード

- `json_loader.py`
- `data_adapter.py`
- `model_registry.py`

### 役割

- JSON を読み込む
- データクラス（KampoFormula, DiseaseMatrix, Phytochemical…）に変換
- エンジンに渡す

### 結果

**データとロジックが完全に分離され、世界標準のアーキテクチャになる。**

---

# 3️⃣ ロジックレイヤー（既存の Python エンジンを整理）

### 目的

あなたが貼ってくれた HealthBook 統合エンジンを、  
**GitHub で動く “推論エンジン” として整理する。**

### 使うAIツール

- Cursor（コード整理・リファクタリング）
- ClaudeCode（補助コード生成）

### ここで行うこと

- HealthBookIntegratedEngine を `/src/healthbook_engine.py` に分離
- MBT55 ロジックを `/src/mbt55_logic.py` に分離
- AgriWare の GHG モデルを `/src/agriware_engine.py` に分離
- SafetyChain のロジックを `/src/safetychain_engine.py` に分離

### 結果

**GitHub 上で “実行可能なエンジン” が完成する。**
