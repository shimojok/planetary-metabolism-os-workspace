## 🧭 1. Antigravity が何者か（Kaz の視点で要点だけ）

Antigravity は Google が開発した **次世代 AI IDE（エージェント型開発環境）** で、  
従来の「AIがコードを補完する」ではなく、

> **AIが“自律的にタスクを実行する”開発環境**  
> （ブラウザ操作・複数リポジトリ管理・テスト生成・成果物作成まで）

という点が最大の特徴です。 [antigravity.google](https://antigravity.google/)

Cursor や Copilot が「会話相手」だとすると、  
Antigravity は **“開発チームの一員として働くAI”** です。

---

## 🚀 2. Kaz のプロジェクトに対して有用な理由（最重要）

あなたのプロジェクトは、一般的なアプリ開発ではなく、

- MBT55（生態学・代謝学モデル）
- AgriWare（GHG・グリーンプレミアム）
- HealthBook（代謝解析）
- SafetyChain（食品安全）
- Probiotics／Herbal／Food スクリーニング
- GitHub でのデモシステム構築
- Azure 実装
- 財団・Microsoft への提出

という **複雑なモデル × 大規模データ × 複数リポジトリ × 文書生成** が必要です。

Antigravity は、まさにこの「複雑性の統合」に強い。

## 🔥 3. Kaz にとっての具体的メリット（検索情報＋あなたの用途で整理）

### ① **複数リポジトリを横断して開発できる**

AgriWare／HealthBook／SafetyChain／MBT55 を  
**1つの IDE で同時に扱える**のは Antigravity の強み。 [Qiita](https://qiita.com/akira_papa_AI/items/0acf2679e4ce9f7fb153)

- `/src/agriware_engine`
- `/src/healthbook_engine`
- `/src/mbt55_logic`
- `/src/safetychain_engine`

などを横断して AI が理解し、変更を提案できる。

---

### ② **ブラウザ操作エージェントが強力**

Antigravity には **Browser Agent** があり、  
AI がブラウザを操作して情報収集・検証を行える。 [Qiita](https://qiita.com/akira_papa_AI/items/0acf2679e4ce9f7fb153)

Kaz の用途では：

- Gates財団・Rockefeller財団・世界銀行・FAO のレポート収集
- Microsoft Azure の API 仕様確認
- 論文・政策文書の取得
- GitHub Actions の設定確認

などを **AIが自動で行える**。

これは Cursor にはない機能。

---

### ③ **Artifacts（成果物）生成が強い**

Antigravity は、コードだけでなく：

- README
- API仕様書
- モデル説明書
- デモ用 Notebook
- Azure 実装ガイド

などの **成果物（Artifacts）を自動生成**できる。 [Qiita](https://qiita.com/akira_papa_AI/items/0acf2679e4ce9f7fb153)

Kaz のプロジェクトは「文書の質」が極めて重要なので、これは大きい。

---

### ④ **VS Code ベースで拡張性が高い**

VS Code の拡張機能がそのまま使えるため、  
あなたの開発環境（Python, JSON, YAML, GitHub, Azure CLI）がそのまま動く。 [Qbook](https://www.qbook.jp/column/2381.html)

---

### ⑤ **AIエージェントが“タスクを完結”させる**

Antigravity は、単なる補完ではなく：

- 設計
- 実装
- テスト
- ブラウザ検証
- 文書生成
- GitHub コミット

まで **AIが自律的に実行**できる。 [Note](https://note.com/kiri_korou/n/nd6471e3f0c87)

Kaz のように「複数プロジェクトを同時に進める」場合、
