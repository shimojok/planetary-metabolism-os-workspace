MBT_Master_part01.json　とは、どのファイルと関連したものですか？

---

結論から言うと、**MBT_Master_part01.json は、あなたが構築している HealthBook／MBT55／漢方代謝ライブラリの “データレイヤー” を構成するファイルであり、Python コード（HealthBook Platform）と密接に関連する “外部データファイル” です。**  
つまり、**HealthBook Platform の中の「漢方処方データ（KampoFormula）」を JSON として外部化したもの**です。

あなたが貼ってくれた Python コードを丁寧に読み解くと、この関係が明確になります。

---

# 1. MBT_Master_part01.json が対応している Python コード部分

あなたの HealthBook Platform には、次のデータクラスがあります。

```python
@dataclass
class KampoFormula:
    id: str
    name: str
    main_herbs: List[str]
    mbt_pathway: MetabolicPathway
    target_metabolite: str
    clinical_indications: List[str]
    agrix_key_minerals: List[str]
```

そして、サンプルとして `_load_kampo_metabolic_library()` に  
F119、F038、F184 などの漢方処方が Python 内にハードコードされています。

本来は 294 方剤あるはずですが、Python コードには **3つしか入っていません**。

---

## MBT_Master_part01.json は、この 294 方剤を外部化したもの

つまり：

- **Python 内に埋め込まれていた漢方データ（KampoFormula）を外部 JSON に分離したもの**
- **294 方剤を複数ファイル（part01, part02, …）に分割したもの**
- **HealthBook Platform のデータベース初期化で読み込むべき “本番データ”**

という位置づけになります。

---

# 2. なぜ part01 / part02 / part03… に分かれているのか？

理由は明確で、**294 方剤を 1 つの JSON にすると巨大になりすぎるから**です。

一般的に：

- part01 → F001〜F030
- part02 → F031〜F060
- …
- part10 → F271〜F294

のように分割されます。

あなたが以前に言及した通り：

> 「MBT_Master_part01.json（F001–F030）  
> MBT_Master_part02.json（F031–F060）  
> …  
> 294 方剤分がある」

これはまさに **KampoFormula のデータベースを JSON 化したもの**です。

---

# 3. MBT_Master_part01.json はどの Python ファイルと関連するか？

### 直接関連するのはこの関数です：

```python
def _load_kampo_metabolic_library(self) -> Dict[str, KampoFormula]:
```

本来はここで：

- JSON ファイルを読み込み
- KampoFormula データクラスに変換
- self.kampo_db に格納

という処理を行うべきです。

しかし、現状のコードでは：

```python
sample_formulas = {
    "F119": KampoFormula(...),
    "F038": KampoFormula(...),
    "F184": KampoFormula(...)
}
```

と **サンプル3件しか入っていません**。

つまり、**MBT_Master_part01.json は、この部分を置き換えるための本番データ**です。
