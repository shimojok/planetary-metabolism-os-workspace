## 2. JSON 化の段取り（PM としての最適ルート）

GitHub での運用を前提に、以下の順番で進めます。

### **Step 1：問診表 PDF → JSON（200項目）**

- セクションごとに分類
    - 食べる生活習慣
    - 動く・休む
    - 環境・体質
    - 身体症状
    - 健診データ
- 各項目に **カテゴリ・リスク因子・代謝影響・疾病リンク・重み** を付与
- GitHub に `/data/questionnaire/questionnaire_v1.json` として保存

### **Step 2：病気と食生活の関係シート → 疾病マトリックス JSON**

- 137 疾病 × 50 食生活因子
- 各疾病に対して
    - リスク因子
    - 重み
    - 代謝経路
    - 推奨食品
    - ファイトケミカル
- GitHub に `/data/disease_matrix/disease_matrix_v1.json`

### **Step 3：ファイトケミカル分類 → JSON**

- 下條式分類体系を完全 JSON 化
- 体系（ポリフェノール群 → フラボノイド → カテキン類 → カテキン）
- 食品源・効能・推奨疾病
- GitHub に `/data/phytochemicals/phytochemicals_v1.json`

### **Step 4：統合データモデルの生成**

- 問診 → リスク因子 → 疾病 → ファイトケミカル
- すべてをリンクした **HealthBook Knowledge Graph** を JSON で構築
- GitHub に `/data/knowledge_graph/healthbook_kg_v1.json`
