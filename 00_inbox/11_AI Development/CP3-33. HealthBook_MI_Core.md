# HealthBook_MI_Core

**Integrated Metabolic Intelligence Core Engine**  
**食生活・生活習慣・症状・健診データ・MBT55・ファイトケミカルを統合した次世代ヘルスインテリジェンス基盤**

---

## 📌 プロジェクト概要

**HealthBook_MI_Core** は、以下の複数領域を統合する **メタボリック・インテリジェンス（MI）エンジン** を構築するためのコアリポジトリです。

- **浜田式 200 問診**（食生活・生活習慣・症状・環境因子）
- **137 疾病マトリックス**（食生活 × 疾病の関係）
- **下條式ファイトケミカル分類**（植物化学成分 × 代謝防御）
- **MBT55 代謝モデル**（Food & Herbal Metabolic Teleodynamics）
- **腸内細菌データ（MBT55菌数など）**
- **健康診断データ（血液・肝機能・脂質・糖代謝・腫瘍マーカー）**

これらを統合し、  
**「食・漢方・菌・生活習慣・症状」から個別化された代謝改善戦略を導く**  
ことを目的としています。

---

## 🎯 目的

### **1. HealthBook（食生活 × 疾病 × 代謝）コアエンジンの構築**

- 200問診の JSON 化
- リスク因子・代謝影響・疾病リンクの構造化
- 疾病マトリックスの数値化
- 健康診断データとの統合

### **2. MBT55（Food & Herbal Metabolic Teleodynamics）との統合**

- 漢方方剤 × 食品 × ファイトケミカル × 代謝経路
- MBT55 菌数データとの連携
- 代謝クラスタリングモデルの構築

### **3. Probiotics Screening（症状 → 菌 → 食・漢方）への拡張**

HealthBook の構造化データを基盤として、  
**症状 → 代謝パターン → 必要な菌 → 食品/漢方/ファイトケミカル**  
というスクリーニングシステムを構築します。

## 🧠 コア概念

### **1. 完全構造化された問診データモデル（Kaz 提案の 3. 完全構造）**

各問診項目は以下の構造で JSON 化します：

```json
{
  "id": 2,
  "category": "食習慣",
  "question": "朝食を抜くことがよくある",
  "risk_factors": ["欠食型（朝食抜き）"],
  "metabolic_impact": "糖新生リズムの乱れ → 午前中の低血糖傾向",
  "disease_links": ["糖尿病", "肥満", "慢性疲労"],
  "weight": 0.85
}
```

## 🔗 統合モデル

### **HealthBook → MBT55 → Probiotics Screening**

このリポジトリは、最終的に以下の統合モデルを実現します：

1. **問診・生活習慣 → 代謝パターン推定**
2. **代謝パターン → 必要な菌（MBT55）推定**
3. **菌 → 食品・漢方・ファイトケミカル推奨**
4. **症状 → 代謝 → 菌 → 食・漢方 → 改善戦略**
