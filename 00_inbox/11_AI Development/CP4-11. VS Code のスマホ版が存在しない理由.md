VS Code のスマホ版はありますか？

---

結論はこうです。

**VS Code の“公式スマホ版”は存在しません。**  
ただし、スマホで VS Code とほぼ同じことを実現する方法は複数あります。  
そして、Kaz のプロジェクト（MBT55 / AgriWare / HealthBook / SafetyChain）に最適な構成も明確にできます。

---

## VS Code のスマホ版が存在しない理由
VS Code はデスクトップ向けに設計されており、  
- 拡張機能  
- GitHub 連携  
- Copilot  
- ターミナル  
- ファイルシステムアクセス  
などが必要なため、スマホアプリとしては提供されていません。

---

## ただし、スマホで VS Code を“使う”方法は3つある

### 1) **VS Code Web（ブラウザ版）**
スマホのブラウザで  
**https://vscode.dev**  
にアクセスすると、VS Code の軽量版が動きます。

- Markdown 編集  
- JSON / YAML  
- 軽いコード編集  
- GitHub 連携（読み書き可能）  
- 拡張機能の一部が動作  

ただし、**GitHub Copilot は動きません**。

---

### 2) **GitHub Codespaces（スマホブラウザで利用可能）**
スマホのブラウザで  
**GitHub Codespaces**  
を開くと、クラウド上の VS Code が動きます。

- GitHub Copilot が動く  
- ターミナルも使える  
- Python / Node / Jupyter も動く  
- Vault を GitHub に置けばスマホで編集可能  

これは **スマホで VS Code を使う最強の方法** です。

---

### 3) **サードパーティの VS Code クローンアプリ**
例：  
- Code App  
- DroidEdit  
- Acode  
など。

ただし、**Copilot は動かない**ため、Kaz の用途には不向き。

---

## Kaz のプロジェクトに最適な構成（スマホ × VS Code × Copilot）

あなたのプロジェクトは：

- MBT55 の理論  
- AgriWare のモデル  
- HealthBook の仕様  
- SafetyChain の構造  
- Planetary OS のエージェント設計  

という「知識 × コード × デモ」が必要。

そのため、スマホでの最適解は次の構成です。

---

## 最適解：**スマホ → GitHub Codespaces → VS Code + Copilot**

### この構成のメリット
- スマホで VS Code が動く  
- GitHub Copilot が使える  
- Obsidian Vault を GitHub に置けば、スマホで編集可能  
- MBT55 Agent のコードもスマホで書ける  
- デモシステムもスマホで動く  

つまり：

> **スマホだけで Planetary OS の開発ができる。**

---

## では、Obsidian Vault（4,700ドキュメント）はどう扱う？

### スマホでの最適ルートはこれ：

1. Obsidian Vault を GitHub にバックアップ  
2. スマホで GitHub Codespaces を開く  
3. Vault を VS Code として読み込む  
4. Copilot がコード補完・API生成を行う  

これで、  
**スマホ → VS Code → Copilot → Vault → GitHub**  
という完全な開発パイプラインが成立します。

---

- VS Code の公式スマホ版は存在しない  
- しかし、**GitHub Codespaces を使えばスマホで VS Code + Copilot が動く**  
- Kaz のプロジェクトにはこの構成が最適  
- Obsidian Vault も GitHub に置けばスマホで編集可能
