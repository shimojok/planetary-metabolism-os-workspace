## ハイパーサイクル指標 H の数式モデル（3段階）

Kaz のプロジェクトに最適化した形で、3つのレベルを用意します。

---

### ① **Level 1：シンプル線形モデル（v0 実装用）**

\[
H = \alpha C_f + \beta D_m + \gamma R_b
\]

- \(C_f\)：炭素固定効率  
- \(D_m\)：微生物多様性（Shannon 指数など）  
- \(R_b\)：Redox 安定性（Eh の変動幅から算出）  

**メリット：**  
- 実装が簡単  
- すぐに動く  
- Notebook で可視化しやすい  

---

### ② **Level 2：循環構造を明示したモデル（v1）**

ハイパーサイクルは「ループの強度」で定義できます。

\[
H = \sqrt[4]{E_{microbe} \cdot E_{nutrient} \cdot E_{plant} \cdot E_{return}}
\]

- \(E_{microbe}\)：微生物代謝活性  
- \(E_{nutrient}\)：栄養カスケード効率  
- \(E_{plant}\)：植物の光合成・根圏分泌  
- \(E_{return}\)：植物→微生物へのフィードバック  

**メリット：**  
- “循環”を数式として表現  
- MBT55 の本質に近い  

---

### ③ **Level 3：非線形創発モデル（v2）**

創発は非線形なので、最終的には：

\[
H = \frac{1}{1 + e^{-k(S - S_0)}}
\]

- \(S\)：代謝ネットワークの結合度（後で定義）  
- \(S_0\)：創発が立ち上がる閾値  
- \(k\)：急峻性  

**メリット：**  
- 「創発の閾値」を表現できる  
- Azure Immune Agent の判断基準に使える

## MBT55 Agent に追加する新しい関数（Python スケルトン）

VS Code にそのまま貼れる形で提示します。

---

### `hypercycle.py`

```python
from .models import SoilState, OrganicInput, MBT55Params
from .carbon_cycle import compute_carbon_fixation_efficiency
from .electron_flow import compute_electron_flow


def compute_hypercycle_index(
    soil: SoilState,
    org: OrganicInput,
    params: MBT55Params,
    diversity_index: float,
    redox_balance: float,
    alpha: float = 0.5,
    beta: float = 0.3,
    gamma: float = 0.2
) -> float:
    """
    ハイパーサイクル指標 H（Level 1 モデル）
    H = α C_f + β D_m + γ R_b
    """

    # 電子フロー → C_f を計算
    je = compute_electron_flow(soil, org, params)
    cf = compute_carbon_fixation_efficiency(je, params)

    # H を計算
    H = alpha * cf + beta * diversity_index + gamma * redox_balance

    # 0〜1 にクリップ
    return max(0.0, min(1.0, H))
```
