結論から言うと **はい、「テストする」という意味です。**  
ただし、ここで言う「テスト」は難しいものではなく、**MBT55 Agent の GHG モデルが正しく動くかを、VS Code で軽く動かして確認する**という意味です。

Kaz がやるべきことは、次の 3 ステップだけです。

---

## 1. VS Code に貼ったコードが動くか確認する  
これは、MBT55 Agent の **v0 が正常に動くか**を見るための最初のチェックです。

例えば、次のような簡単なテストコードを作ります。

```python
from mbt55_agent.models import SoilState, OrganicInput, MBT55Params
from mbt55_agent.ghg import estimate_ghg_changes

soil = SoilState(
    soc=20.0,
    cn_ratio=12.0,
    ph=6.8,
    moisture=55.0,
    temperature=25.0
)

org = OrganicInput(
    carbon=5.0,
    nitrogen=0.3,
    lignin_ratio=0.2,
    lipid_ratio=0.1,
    protein_ratio=0.3,
    starch_ratio=0.4
)

params = MBT55Params(
    kd=0.1,
    lambda_cf=0.8,
    km_ed=10.0,
    eta=0.5,
    guild_activity={
        "starch": 0.8,
        "protein": 0.7,
        "lipid": 0.6,
        "cellulose": 0.9
    }
)

print(estimate_ghg_changes(soil, org, params))
```

これを実行すると、次のような出力が返ってきます（例）：

```
{
  "delta_co2": -3.1,
  "delta_ch4": -0.02,
  "delta_n2o": -0.004
}
```

このように **数値が返ってくれば成功**です。

---

## 2. 数値の「傾向」を見る  
v0 の目的は「正しい傾向が出るか」です。

- SOC が増える → CO₂ が減る  
- 電子フローが大きい → CH₄ が減る  
- Redox が安定 → N₂O が減る  

この方向性が出ていれば、モデルは正しく動いています。

---

## 3. その結果を見て、次の調整を行う  
Kaz の実感や AGRIX のデータに合わせて、係数を調整していきます。

- 0.7  
- 0.01  
- 0.005  
- 3.67  

などの係数は、後で **MBT55 の実データに合わせてキャリブレーション**できます。
