### 1. MBT 漢方代謝ライブラリー：データ構造と因果グラフ

#### データ構造（主なテーブル／エンティティ）

- **生薬**
    - ID, 名称（和名／中名／ラテン名）, 部位, 性味, 帰経, 主成分
- **成分**
    - ID, 名称, 化学構造, 代謝経路ID, 作用標的
- **代謝経路**
    - ID, 経路名, 関与酵素, 生成物／中間代謝物
- **微生物（腸内・土壌）**
    - ID, 属・種, 代謝能, 関連成分, 関連疾患／機能
- **体質・パターン**
    - ID, 体質分類, 症候群, 関連生薬／処方
- **アウトカム**
    - ID, 指標（炎症、血糖、脂質、免疫など）, 測定方法

#### 因果グラフ（概念）

- 生薬 → 成分 → 代謝経路 → 代謝物 → 生理作用 → アウトカム
- 微生物 → 成分変換／代謝 → 代謝物 → アウトカム
- 体質 → 生薬選択 → 作用プロファイル → アウトカム

---

### 2. HealthBook Platform：データ構造と因果グラフ

#### データ構造

- **個人プロファイル**
    - ID, 年齢, 性別, 生活環境, 職業
- **ライフログ**
    - 食事（内容・時間）, 睡眠, 活動量, ストレス指標
- **バイオマーカー**
    - 血液検査, 代謝指標, 炎症マーカー, ホルモン
- **マイクロバイオーム**
    - 微生物プロファイル（相対量）, 多様性指標
- **健康アウトカム**
    - 疾患有無, リスクスコア, 自覚症状, QOL指標

#### 因果グラフ（概念）

- 食事／生活習慣 → 微生物 → 代謝 → バイオマーカー → 疾患リスク
- 環境（騒音・大気・ストレス） → 免疫／ホルモン → アウトカム
- 介入（食事変更・生薬・プロバイオティクス） → 微生物／代謝 → アウトカム

---

### 3. AGRIX：データ構造と因果グラフ

#### データ構造

- **圃場／ロット**
    - ID, 位置情報, 土壌タイプ, 気候帯, 所有者
- **土壌データ**
    - 有機炭素, 窒素, pH, 水分, 微生物プロファイル, 物理性
- **作物・栽培情報**
    - 作物種, 品種, 作付け日, 施肥, 農薬, 農法（慣行／再生型など）
- **環境データ**
    - 気温, 降水, 日射, 風, 極端気象
- **収量・品質**
    - 収量, 等級, 成分（栄養・機能性）
- **炭素・環境指標**
    - 土壌炭素変化, 排出量, 生物多様性指標

#### 因果グラフ（概念）

- 土壌条件／微生物 → 作物生育 → 収量／品質
- 農法（耕起／不耕起／カバークロップ） → 土壌炭素／微生物 → 生態系 → 収量／レジリエンス
- 気候 → 水分／ストレス → 収量／リスク
- 農法 → 炭素フロー → 炭素クレジットポテンシャル

### 4. 3つをAIの「知識データベース＋推論土台」にするイメージ

- **知識データベースとして：**
    
    - これらのスキーマをグラフDB（Knowledge Graph）として構造化
    - ノード＝生薬／成分／微生物／土壌／作物／人／政策…
    - エッジ＝因果・関連・所属・位置づけ
- **推論エンジンの土台として：**
    
    - 因果グラフ（DAG）として「どこに介入すると何が変わるか」を定義
    - 生成AI（LLM）は：
        - ユーザーの質問 → グラフ上のクエリに変換
        - 結果を自然言語で説明
    - 将来的には因果推論アルゴリズム（do-calculus 等）と接続
