## 各Dayで何をするか

### Day3（今日）: 分類エージェント

**目的**: 500件のノートに「これはMBT55の話か、AGRIXか」というラベルを貼る

```
pre_filtered.json（500件）
        ↓
classifier.py が各ノートを読んで判定
        ↓
ドメイン分類結果 + エンティティ抽出
（例: MBT55 / AGRIX / HealthBook / PBPE / general）
        ↓
classified_notes.json
```

**所要時間**: API呼び出しが500件あるので、実行中は30〜60分かかります。途中で止まっても再開できます。

---

### Day4: 批評エージェント

**目的**: 分類結果の品質チェック。「本当にこの分類は正しいか」を自動検証する

```
classified_notes.json
        ↓
critic.py が各分類を検証
quality_score >= 0.85 なら承認
quality_score < 0.85 なら再分類（最大3回）
        ↓
approved_notes.json（品質保証済み）
```

**ポイント**: ここで弾かれたノートは使いません。品質を妥協しない。

---

### Day5: 承認済みノート確定

**目的**: approved_notes.jsonの中身を人間の目で確認する

```
approved_notes.json
        ↓
check_notes.py で統計確認
・何件承認されたか
・ドメイン別の内訳は適切か
・スコアの分布はおかしくないか
        ↓
問題なければWeek2へ進む
問題あれば閾値を調整して再実行
```

**これが唯一の手作業です。** 数字を見て判断するだけです。

---

### Day6-7: Week2の準備

**目的**: Neo4jをDockerで起動して接続確認

```
Docker Desktop起動

        ↓
接続テスト（ブラウザで localhost:7474 を開く）
        ↓

```
