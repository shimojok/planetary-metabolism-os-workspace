# PMOS Ontology（実装設計）

まず結論から。

PMOS Ontologyは以下の**5階層構造**で設計します。

```
Layer 0: Universe（全体概念）
Layer 1: Domain（領域）
Layer 2: System（システム）
Layer 3: Entity（実体）
Layer 4: Process / Function（作用）
```

---

# 1. Layer 0：Universe（最上位）

中心概念は一つです。

```
Planetary Metabolism
```

Kazさんの全プロジェクト

* AGRIX
* MBT55
* HealthBook

はすべてこの下に統合されます。

---

# 2. Layer 1：Domain（研究領域）

6領域に固定します。

```
Soil System
Food System
Microbiome System
Human System
Climate System
Economic System
```

重要ポイント：

これは単なる分類ではなく

**代謝の連鎖構造**

です。

```
Soil → Food → Microbiome → Human → Economy → Climate
```

---

# 3. Layer 2：System（機能系）

各Domainの中に「機能単位」を定義します。

例：

### Soil Domain

```
Soil Carbon Cycle
Soil Microbiome Network
Nutrient Cycling System
```

### Human Domain

```
Metabolic Regulation System
Immune System
Endocrine System
```

---

# 4. Layer 3：Entity（ノード）

ここが**Knowledge Graphのノード**になります。

例：

```
Soil Microbiome
Bacillus subtilis
Short Chain Fatty Acids
Insulin Sensitivity
CO2 Emissions
```

重要：

Entityは必ず

```
測定可能 or 定義可能
```

である必要があります。

---

# 5. Layer 4：Process / Function（エッジ）

ここが**因果関係（Edges）**です。

例

```
produces
increases
decreases
regulates
transforms
```

例：

```
Soil Microbiome
  ──produces──▶ Nutrients

Nutrients
  ──increase──▶ Gut Microbiome Diversity

Gut Microbiome
  ──regulates──▶ Metabolic Health
```

---

# 6. Ontology JSON設計

実装用フォーマットです。

```json
{
  "ontology": {
    "universe": "Planetary Metabolism",

    "domains": [
      "soil",
      "food",
      "microbiome",
      "human",
      "climate",
      "economy"
    ],

    "entity_types": [
      "organism",
      "compound",
      "process",
      "system",
      "metric"
    ],

    "relation_types": [
      "produces",
      "consumes",
      "increases",
      "decreases",
      "regulates",
      "transforms",
      "emits",
      "absorbs"
    ]
  }
}
```
