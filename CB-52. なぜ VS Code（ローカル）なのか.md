# ■ なぜ VS Code（ローカル）なのか

## ① Vault（Obsidian）と直接連携できる

- ローカルにある `.md` をそのまま読み込める
    
- 4500ノート規模でも処理可能
    

👉 クラウドだと毎回アップロードが必要 → 非効率

---

## ② ファイル生成（自動保存）が高速

今回やる処理：

```text
読み込み → 分割 → ノード抽出 → 保存
```

👉 すべてローカルI/Oの方が圧倒的に速い

---

## ③ Neo4j / Docker と連携しやすい

後のフェーズで：

```text
Python → Neo4j（ローカル）
```

👉 VS Code環境が最も自然

---

## ④ 開発・修正ループが速い

- コード修正 → 即実行
    
- デバッグ可能

# ■ 実際の作業手順（迷わない手順）

## ① VS Codeでプロジェクトフォルダ作成

```bash

```

---

## ② VS Codeで開く

```text
File → Open Folder → MBT55_Project
```

---

## ③ Python環境確認

```bash
python --version
```

👉 3.10以上ならOK

---

## ④ 仮想環境（推奨）

```bash
python -m venv venv
```

起動：

```bash
# Windows
venv\\Scripts\\activate

# Mac/Linux
source venv/bin/activate
```

---

## ⑤ ライブラリインストール

```bash
pip install openai langchain chromadb networkx python-dotenv neo4j
```

---

