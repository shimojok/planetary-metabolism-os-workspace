仮想環境が有効化されていない状態でpipを実行しようとしています。以下の手順で解決できます：

## 解決手順

### ① まず仮想環境を有効化

```powershell
# プロジェクトフォルダにいることを確認
cd PMOS_12day

# 仮想環境を有効化
.\venv\Scripts\activate

# 成功すると、プロンプトの先頭に (venv) と表示されます
# 例: (venv) PS C:\Users\Kaz\PMOS_12day>
```

### ② 仮想環境が有効化された状態で再実行

```powershell
# これで正常に動きます
pip install -r requirements.txt
```

### ③ もし「pip」がまだ認識されない場合

```powershell
# Python自体がインストールされているか確認
python --version

# Pythonが入っていれば、こちらでもOK
python -m pip install -r requirements.txt
```

### ④ それでも動かない場合（Pythonのパス問題）

```powershell
# Pythonのインストール場所を確認
where python

# 例: C:\Users\Kaz\AppData\Local\Programs\Python\Python310\python.exe

# フルパスで実行
C:\Users\Kaz\AppData\Local\Programs\Python\Python310\python.exe -m pip install -r requirements.txt
```

## 仮想環境の基本的な使い方（参考）

```powershell
# 仮想環境を作成（一度だけ）
python -m venv venv

# 仮想環境を有効化（毎回作業開始時）
.\venv\Scripts\activate

# 仮想環境から出る時
deactivate
```

**毎回作業を始める時は、必ず `.\venv\Scripts\activate` を実行してから pip コマンドを使います。**

今、仮想環境を有効化してから再実行してみてください。
